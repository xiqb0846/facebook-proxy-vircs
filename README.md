## 静态独享 vs 动态轮转：Facebook运营该选哪种

两种代理类型适合完全不同的场景，混用是最大的坑。

**动态轮转IP**：每次请求换一个IP，适合大规模数据采集、短期测试。用来养Facebook账号？基本等于慢性自杀——Facebook会记录你的IP历史，频繁切换只会让系统判定行为异常。

**静态独享IP**：一个账号绑定一个IP，长期稳定，独自使用没有共享污染。这才是Facebook账号运营的正确打开方式。一个美国家庭住宅静态IP，配一个Facebook账号，就是在模拟"这个账号的主人是一个住在美国的真实用户"。

结论很简单：**养号选静态独享，采集选动态轮转，不要混用。**
